
Este proyecto de gestión universitaria permite registrar estudiantes y materias, gestionar notas y visualizar la información desde una interfaz web o desde la terminal. Todo lo registrado se almacena en una base de datos SQL Server y se refleja en tiempo real en ambos entornos.

Explicación de cada archivo y sección principal:

ConsoleMenu.java: Archivo Java que muestra un menú interactivo en la terminal. Permite al usuario registrar estudiantes y materias desde la línea de comandos. Cada opción del menú llama a funciones que envían los datos al backend.

MenuActions.java: Contiene los métodos que realizan las acciones del menú de consola. Cada método solicita los datos al usuario por consola y los envía al backend usando peticiones HTTP (POST) para que se almacenen en la base de datos.

GestionUniversitariaApplication.java**: Es la clase principal de la aplicación Spring Boot. Su función es iniciar el servidor backend y cargar todos los componentes necesarios.

controller : Carpeta que contiene los controladores REST (`EstudianteController.java` y `MateriaController.java`). Estos archivos exponen los endpoints de la API REST para crear, listar y actualizar estudiantes y materias. Permiten que tanto el frontend web como el menú de consola interactúen con la base de datos a través de HTTP.

model : Carpeta con las entidades JPA (`Estudiante.java` y `Materia.java`). Cada clase representa una tabla en la base de datos y define los campos que se almacenan para cada estudiante o materia.

repository : Carpeta con las interfaces JPA (`EstudianteRepository.java` y `MateriaRepository.java`). Permiten acceder y manipular los datos de estudiantes y materias en la base de datos de forma sencilla usando Spring Data JPA.

application.properties : Archivo de configuración donde se define la conexión a la base de datos SQL Server (usuario, contraseña, URL, dialecto, etc). Permite que la aplicación se conecte y guarde los datos de forma persistente.

index.html : Archivo HTML que implementa la interfaz web del sistema. Permite a los usuarios y administradores registrar estudiantes y materias, modificar notas y ver toda la información en tiempo real. Incluye formularios, tablas y paneles diferenciados para cada tipo de usuario.

pom.xml : Archivo de configuración de Maven. Define todas las dependencias necesarias (Spring Boot, JPA, SQL Server, etc.) y la configuración del proyecto para compilar y ejecutar correctamente.

El backend expone una API REST que es consumida tanto por el frontend web como por el menú de consola. Los cambios realizados desde cualquier interfaz se reflejan en tiempo real en la otra.


Comandos de terminal para ejecutar el sistema

Lista de comandos útiles para compilar y ejecutar cada parte del sistema, con su explicación:

 mvn clean package  → Compila el proyecto y descarga todas las dependencias necesarias.
 mvn spring-boot:run   → Inicia el backend de Spring Boot y expone la API REST.
 javac -cp ".;target/classes" src/main/java/com/gestionuniversitaria/ConsoleMenu.java src/main/java/com/gestionuniversitaria/MenuActions.java  → Compila los archivos del menú de consola.
 java -cp ".;target/classes;src/main/java" com.gestionuniversitaria.ConsoleMenu`  → Ejecuta el menú de consola para registrar estudiantes y materias desde la terminal.

Cada comando debe ejecutarse en la raíz del proyecto y permite poner en marcha el backend y el menú de consola para interactuar con el sistema.
