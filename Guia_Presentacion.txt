GUÍA DE PRESENTACIÓN - GESTIÓN UNIVERSITARIA

1. INTRODUCCIÓN
Este proyecto es un sistema de gestión universitaria desarrollado en Java que permite administrar Estudiantes, Profesores, Cursos, Inscripciones y Calificaciones.
Utiliza una arquitectura moderna y eficiente, separando el Backend (Lógica) del Frontend (Interfaz de Usuario).

2. TECNOLOGÍAS UTILIZADAS
- Backend: Java con Javalin (Framework web ligero y rápido).
- Frontend: HTML5, JavaScript y Bootstrap 5 (Diseño responsivo).
- Datos: Almacenamiento en archivo JSON (Persistencia de datos).
- Comunicación: API REST y WebSockets (Para actualizaciones en tiempo real).
- Construcción: Maven (Gestión de dependencias y empaquetado).

3. EXPLICACIÓN DEL CÓDIGO (ESTRUCTURA)

A. src/main/java/app/Main.java
   - Es el punto de entrada de la aplicación.
   - Configura el servidor web en el puerto 7000.
   - Define las rutas de la API (ej: /api/estudiantes) y las conecta con los Controladores.
   - Inicia el servicio de WebSockets para la comunicación en tiempo real.

B. src/main/java/controllers/
   - Contiene la lógica de negocio dividida por módulos:
     * EstudianteController: Gestiona la creación, lectura, actualización y eliminación de estudiantes.
     * ProfesorController: Lo mismo para profesores.
     * CursoController, InscripcionController, CalificacionController: Gestionan sus respectivas entidades.
     * ReporteController: Genera los boletines de notas y resúmenes.
   - Controlador.java: Maneja la lógica de los WebSockets para notificar a todos los usuarios cuando hay cambios.

C. src/main/java/utils/DataStore.java
   - Es el "corazón" de los datos.
   - Utiliza Mapas (HashMaps) para almacenar la información en memoria, lo que hace que las búsquedas sean instantáneas (O(1)).
   - Se encarga de guardar y cargar todo en el archivo 'data.json'.

D. src/main/resources/public/index.html
   - Es la interfaz gráfica del usuario.
   - Es una "Single Page Application" (SPA), lo que significa que no recarga la página al navegar.
   - Se conecta al Backend mediante 'fetch' (para pedir datos) y 'WebSocket' (para recibir actualizaciones automáticas).

4. PUNTOS CLAVE A DESTACAR (PARA EL PROFESOR)
- Optimización: Se cambiaron las Listas por Mapas para mejorar el rendimiento de búsqueda de O(n) a O(1).
- Tiempo Real: El uso de WebSockets permite que si un administrador agrega un estudiante, todos los demás administradores lo vean al instante sin recargar.
- Modularidad: El código está organizado en paquetes (controllers, models, utils) siguiendo buenas prácticas.
- Empaquetado: Todo el proyecto (código y web) se compila en un único archivo JAR ejecutable.

5. CÓMO EJECUTAR EL PROYECTO

Paso 1: Construir el proyecto (Compilar)
Abre la terminal en la carpeta del proyecto y ejecuta:
mvn clean package

Paso 2: Ejecutar la aplicación
Una vez termine de compilar, ejecuta:
java -jar target/gestion-universitaria-1.0.jar

Paso 3: Usar la aplicación
- Abre tu navegador en: http://localhost:7000
